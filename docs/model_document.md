# 牛New寿喜烧自助餐排队等待时间模型文档

## 1. 问题背景

餐厅营业用餐时间为 16:00-22:00。记 16:00 为 \(t=0\)，单位为分钟，因此 22:00 对应 \(t=360\)。餐厅共有 \(n\) 张桌子，每桌最多用餐 120 分钟。顾客可以提前取号，但只能从 16:00 开始入场。

核心运营假设是：餐厅 16:00 后基本持续满座；只要有桌子空出，下一桌排队顾客立即入场。因此，某位顾客的等待时间主要由未来桌子释放事件决定。

## 2. 变量定义

- \(c\)：当前时刻，单位为从 16:00 开始的分钟数。
- \(n\)：餐桌总数。
- \(a_j\)：当前时刻第 \(j\) 张桌子已经用餐的分钟数。
- \(X\)：自然用餐时间，即没有 120 分钟限制时的潜在用餐时间。
- \(T=\min(X,120)\)：实际占桌时间。
- \(S_T(t)=P(T>t)\)：实际占桌时间的 survival function。
- \(R_j(c)\)：当前时刻第 \(j\) 张桌子的剩余用餐时间。
- \(D_{j,m}\)：从当前时刻 \(c\) 开始，第 \(j\) 张桌子的第 \(m\) 次未来释放时间。
- \(D_{(r)}(c)\)：从当前时刻 \(c\) 开始，所有桌子合并后的第 \(r\) 次总体释放时间。
- \(k\)：目标顾客前方等待的桌数。
- \(W_k(c)\)：目标顾客等待时间。
- \(A_k(c)\)：目标顾客进场时间。

## 3. 用餐时间截断生存模型

设自然用餐时间 \(X\) 服从 Weibull 或 Lognormal 分布。实际用餐时间受餐厅规则限制：

\[
T=\min(X,120).
\]

因此，当 \(0 \le t < 120\) 时：

\[
F_T(t)=P(T \le t)=P(X \le t)=F_X(t),
\]

\[
S_T(t)=P(T>t)=P(X>t)=S_X(t).
\]

在 \(t=120\) 处，\(T\) 有一个点质量：

\[
P(T=120)=P(X \ge 120)=S_X(120).
\]

这表示用满 2 小时的顾客。注意本文代码中 `survival(120)=0`，因为实际随机变量 \(T\) 不会超过 120；点质量通过采样和 `prob_full_duration()` 表达。

## 4. 条件剩余寿命分布

如果当前时刻第 \(j\) 张桌子已用餐 \(a_j\) 分钟，并且它仍被占用，则其剩余时间 \(R_j(c)\) 满足：

\[
P(R_j(c)>r \mid T>a_j)
=P(T-a_j>r \mid T>a_j)
=\frac{S_T(a_j+r)}{S_T(a_j)},
\]

其中：

\[
0 \le r \le 120-a_j.
\]

当 \(r=120-a_j\) 时，剩余时间不可能更大，因此 survival 为 0。由于 \(T\) 在 120 分钟处有点质量，\(R_j(c)\) 在 \(120-a_j\) 处也可能有点质量。

## 5. 多桌释放过程

对第 \(j\) 张桌子，其未来释放时间序列定义为：

\[
D_{j,1}=R_j(c),
\]

\[
D_{j,2}=R_j(c)+T_{j,1},
\]

\[
D_{j,3}=R_j(c)+T_{j,1}+T_{j,2}.
\]

一般地：

\[
D_{j,m}=R_j(c)+\sum_{\ell=1}^{m-1}T_{j,\ell},
\]

其中 \(T_{j,\ell}\) 是第 \(j\) 张桌子在当前顾客离席后，第 \(\ell\) 批新顾客的实际用餐时间。模型假设这些未来用餐时间独立同分布，并与当前桌子剩余时间独立。

## 6. 第 \(k+1\) 个释放事件与等待时间预测

将所有桌子的未来释放时间合并并排序：

\[
D_{(1)}(c)\le D_{(2)}(c)\le D_{(3)}(c)\le \cdots.
\]

这里 \(D_{(r)}(c)\) 表示从当前时刻 \(c\) 开始的第 \(r\) 次总体释放时间。

如果目标顾客前方还有 \(k\) 桌，那么前 \(k\) 次释放会被前方顾客使用，目标顾客在第 \(k+1\) 次释放时入场。因此：

\[
W_k(c)=D_{(k+1)}(c),
\]

\[
A_k(c)=c+D_{(k+1)}(c).
\]

这是本项目中等待时间预测的核心关系。

## 7. Monte Carlo 计算方法

每次 Monte Carlo 重复执行：

1. 对每张当前被占用的桌子，根据条件剩余寿命分布采样 \(R_j(c)\)。
2. 对每张桌子的后续顾客采样 iid 的 \(T_{j,\ell}\)。
3. 用最小堆合并所有桌子的未来释放事件，直到得到第 \(k+1\) 个总体释放事件。
4. 记录 \(W_k(c)=D_{(k+1)}(c)\)。

重复 \(B\) 次后，得到等待时间样本：

\[
W_k^{(1)}(c),\ldots,W_k^{(B)}(c).
\]

由这些样本计算均值、中位数、分位数和概率：

\[
P(W \le 30),\quad P(W \le 60),\quad P(W \le 90).
\]

进场时间样本为：

\[
A_k^{(b)}(c)=c+W_k^{(b)}(c).
\]

## 8. 输入输出说明

核心函数为：

```python
predict_entry_time(
    current_time,
    queue_ahead,
    n_tables,
    dining_time_distribution,
    table_ages=None,
    n_simulations=10000,
    random_state=None,
)
```

输入：

- `current_time`：当前时刻，单位为从 16:00 开始的分钟数。
- `queue_ahead`：前方还有多少桌。
- `n_tables`：餐桌数。
- `dining_time_distribution`：用餐时间分布对象。
- `table_ages`：当前每张桌子的已用餐时间。如果缺失，则使用平稳年龄近似。
- `n_simulations`：Monte Carlo 次数。

输出：

- `mean_wait`
- `median_wait`
- `q10_wait`
- `q90_wait`
- `p_wait_le_30`
- `p_wait_le_60`
- `p_wait_le_90`
- `mean_entry_time`
- `median_entry_time`
- `q10_entry_time`
- `q90_entry_time`
- `waiting_time_samples`
- `entry_time_samples`

### table_ages 缺失时的近似

如果没有当前每张桌子的已用餐时间，代码使用持续满座续订过程的平稳年龄分布近似。直观上，随机观察一个正在进行的用餐区间时，较长的用餐区间更容易被观察到。因此实现中先从 length-biased interval distribution 抽取当前占桌区间长度，再在该区间上均匀抽取当前年龄。

这个近似依赖两个条件：

1. 餐厅在当前时间附近基本持续满座；
2. 当前桌子的用餐过程已接近稳定状态。

在刚开门后不久，或者存在明显空桌时，该近似会偏离真实状态，应尽量输入真实 `table_ages`。

## 9. 如何使用真实历史数据估计 Weibull 参数

如果有历史翻台数据，记录每桌实际离席时间。对未用满 120 分钟主动离席的样本，设：

\[
\delta_i=1.
\]

对用满 120 分钟的样本，只知道自然用餐时间 \(X_i \ge 120\)，视为右删失，设：

\[
\delta_i=0.
\]

Weibull 模型下似然为：

\[
L(\theta,\lambda)=
\prod_i f_X(t_i;\theta,\lambda)^{\delta_i}
S_X(120;\theta,\lambda)^{1-\delta_i}.
\]

代码中的 `estimate_weibull_mle_from_censored_data(times, event_observed)` 使用该删失似然进行最大似然估计，并返回 `TruncatedWeibullDiningTime`。

## 10. 模型局限性

- 当前版本假设取号流均匀到达，没有显式建模晚餐高峰。
- 假设所有桌型相同，没有区分 2 人桌、4 人桌、大桌或拼桌策略。
- 假设只要有桌子空出，下一桌立即入场，没有清台时间。
- 没有建模过号、no-show、顾客临时放弃等行为。
- 当 `table_ages=None` 时使用平稳近似，在刚开门或客流不稳定时可能不准确。
- 参数不确定性尚未进入预测分布；当前预测默认分布参数已知。

## 11. 后续可扩展方向

- 引入非齐次 Poisson 或分段到达率模型，刻画不同时间段取号强度。
- 按桌型和人数建模，让等待时间和桌型匹配相关。
- 加入清台时间、叫号响应时间、过号规则。
- 用 Bayesian survival model 表达参数不确定性。
- 用真实历史数据校准模型并做 posterior predictive check。
- 加入在线更新机制，根据当天已观测的翻台情况动态修正预测。
